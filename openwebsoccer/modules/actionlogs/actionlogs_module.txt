Modul								actionlogs
Admincenter							Einstellungen -> TLC-Features -> Benutzeraktivitäten ausblenen	// eigene Benutzeraktivitäten werden weiterhin angezeigt
Funktionen sichtbar					?page=home			-> Benutzeraktivitäten		// alle Benutzeraktivitäten der letzten Einträge anzeigen
									?page=user&id=		-> Benutzer					// nur die Benutzeraktivitäten des Users anzeigen
									?page=allaktivities	-> Alle User Aktivitäten	// nur für den Admin ( wenn er auch Benutzer angelegt wurde )
Neu									Bei den Benutzeraktivitäten wird nun zusätzlich die Uhrzeit angezeigt
Neu									Neue Seite Im Frontend unter Ligen der Punkt "Alle Useraktivitäten"
									User erhalten die Meldung, das der Zugriff nur für den Admin ist.
									Der Admin erhält eine Liste aller gespeicherten Useraktivitäten.
									Die Voraussetzung ist, das der Admin zusätzlich als Benutzter registriert ist.
									Dabei muss der Benutzername und die E-Mail-Adresse bei beiden Accounts gleich sein !
<page>  Neu							Bereitstellung aller User-Aktivitäten für den Admin
									template="user-activities-all.twig"
									model="AllActivitiesModel.class.php"
<block>								Anzeige aller Useraktivitäten als Liste
									template="user-activities-list.twig"
									model="UserActivitiesModel.php"
<block>								Anzeige der letzten Useraktivitäten in einer Box
									template="user-activities-box.twig"
									model="AllUserActivitiesModel.php"
Datenservice						für die Aktivitäteneinträge
									ActionLogDataService.class.php
Konfiguration-Datei					module.xml
Nachrichten-Datei					messages_de.xml
Hilfsdatei							mehrfach genutzte Komponente für ActionLogDataService.class.php
									columns.php
									logs.php
Klasse mit mehreren Funktionen
ActionLogDataService.class.php		class		ActionLogDataService::
												function				getActionLogsOfUser			// Funktion: Letzte Aktionen eines User
												function				getLatestActionLogs			// Funktion: Letzte 10 Aktionen aller User
												function				getAllActionLogs			// Funktion: Alle Aktionen aller User im verfügbaren Zeitraum
												function				createOrUpdateActionLog		// Funktionen: NeueLogeinträge erstellen, Alte Einträge löschen, bei gleicher Aktion die Zeit korrigieren
Klasssen die auf "implements IModel" basieren - "implements IModel" muss nicht genutzt werden.
Es dient nur zur Sicherung der Mindestvorgaben der Funktionsaufrufe als eigene Klasse.
Klassen mit einer Function
AllActivitiesModel.class.php		class		AllActivitiesModel
												function				__construct
												function				renderView
												function				getTemplateParameters
												return ["activities" => ActionLogDataService::getAllActionLogs($this->_websoccer, $this->_db)];
UserActivitiesModel.php				class		UserActivitiesModel
												function				__construct
												function				renderView
												function				getTemplateParameters
												return ["activities" => ActionLogDataService::getActionLogsOfUser($this->_websoccer, $this->_db, $this->_websoccer->getRequestParameter('userid'))];
AllUserActivitiesModel.php			class		AllUserActivitiesModel
												function				__construct
												function				renderView
												function				getTemplateParameters
												return ["activities" => ActionLogDataService::getLatestActionLogs($this->_websoccer, $this->_db, 5)];
